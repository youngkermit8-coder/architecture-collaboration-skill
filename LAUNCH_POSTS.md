## Reddit

Suggested title:

```text
I built a Codex skill to keep long AI coding projects from losing their architecture
```

Suggested body:

```text
I kept running into the same problem with AI-assisted development:

The first few sessions are clear. The architecture makes sense. The product direction is sharp.

Then the project grows. The chat gets long. Decisions change. The original architecture becomes hard to trace. A later session starts solving locally, but the system-level picture slowly drifts.

So I built a small Codex skill called Architecture Collaboration.

It gives Codex a reusable project memory workflow:

- PROJECT_ARCHITECTURE.md as the concise source of truth
- ARCHITECTURE_CHANGELOG.md for meaningful changes
- SESSION_SUMMARY.md for handoff between sessions
- ADR files for architectural decisions
- versions/ snapshots for traceability

The goal is not to replace project management tools. It is more like a lightweight operating ritual for long-running AI coding projects.

Repo:
https://github.com/youngkermit8-coder/architecture-collaboration-skill

I would love feedback from anyone building with Codex, Claude Code, Cursor, or other agentic coding tools. The main question I am exploring is:

What is the minimum project memory structure an AI coding agent needs to stay useful across many sessions?
```

## Indie Hackers

Suggested title:

```text
I built a tiny project memory system for AI-assisted development
```

Suggested body:

```text
I have been building with AI coding agents and noticed a recurring problem:

AI is fast inside one session, but long-running projects need memory discipline.

If decisions only live in chat history, the project slowly loses its original architecture. You start with a clean system idea, then weeks later you are patching local details without a shared source of truth.

So I built Architecture Collaboration, a Codex skill that gives every project a small memory system:

- a concise architecture file
- a changelog
- a session handoff summary
- ADRs for important decisions
- version snapshots

The interesting part is that each project stays independent, while the workflow itself is reusable.

Repo:
https://github.com/youngkermit8-coder/architecture-collaboration-skill

I am using it for my own product research and long-form agent collaboration. Curious if other builders are running into the same "AI forgets the architecture" problem.
```
